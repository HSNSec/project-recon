# Project-Recon

This project consists of lightweight tools that will automate the reconnaissance phase of a penetration test. All the tools in this project are written in Bash and will be regularly improved and expanded over time.

|**live-hosts Version 1.0**|
|-|



This was just the sketch of the script in order to understand how the output generated by a ping command can be utilised to identify the status of the target hosts.



There are not issues with syntax and will run correctly. However, this script has few gaps which are below:



* The Scans only 1 IP/Domain at a time.
* The output has a lot of noise. Since "ping" command sends ICMP packets to the target, it generates information i.e. packets sent, lost, ttl, etc. We don't need these details for now. If the output is just "Host is up!" or "Host is down!" That's what we need.



Above improvements will be made in **live-hosts Version 1.1**
